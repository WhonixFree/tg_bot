2. Authentication & Request Signing
2.1. Headers

Every API request (except incoming webhooks) must contain:
Header	Type	Req.	Description
Content-Type	string	yes	Always application/json
project	string	yes	Merchant project UUID
sign	string	yes	HMAC-SHA256 request signature
2.2. Signature Algorithm (API Requests, POST)

1. Serialize the request body to JSON.

2. Encode the JSON in Base64.

3. Compute HMAC-SHA256 of the Base64 string using API_KEY.

function apiSign(array $data, string $apiKey): string {
    $json = json_encode($data, JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES);
    $base64 = base64_encode($json);
    return hash_hmac('sha256', $base64, $apiKey);
}

2.3. Signature for Bodyless Requests (GET)

For requests with empty body (e.g., GET /v1/payout/status/{uuid}):

$sign = hash_hmac('sha256', base64_encode(''), $apiKey);

5. Webhook Notifications

The system sends a webhook to `url_callback` when a payment status changes.
Request Format

- Method: POST

- Content-Type: application/json

- Signature: sign field in the request body

{
  "uuid": "abc123-def456-...",
  "order_id": "ORDER-12345",
  "amount": "100.00",
  "currency": "USD",
  "amount_usd": "100.00",
  "exchange_rate": null,
  "url": "https://2328.io/pay/abc123-def456-...",
  "expires_at": "2026-01-11T21:00:00Z",
  "created_at": "2026-01-11T20:00:00Z",
  "payer_currency": "USDT",
  "payer_amount": "100.50",
  "network": "TRX-TRC20",
  "address": "TXYZabc123...",
  "payment_status": "paid",
  "txid": "0xabc123def456...",
  "payment_amount": "100.50",
  "sign": "a1b2c3d4e5f6..."
}

Important: The webhook structure is identical to the /v1/payment/info response, plus a sign field for authentication.
Verify Webhook Signature

1. Extract the sign field from webhook data.

2. Remove the sign field from the array.

3. Encode remaining fields as JSON.

4. Encode JSON in Base64.

5. Compute HMAC-SHA256 from Base64 using API_KEY.

6. Compare the computed signature with sign from the webhook.

<?php
function verifyWebhookSign(array $data, string $apiKey): bool {
    $receivedSign = $data['sign'] ?? '';
    unset($data['sign']);

    $json = json_encode($data, JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES);
    $base64 = base64_encode($json);
    $calculatedSign = hash_hmac('sha256', $base64, $apiKey);

    return hash_equals($calculatedSign, $receivedSign);
}

$apiKey = 'YOUR_API_KEY';
$webhookData = json_decode(file_get_contents('php://input'), true);

if (!verifyWebhookSign($webhookData, $apiKey)) {
    http_response_code(401);
    echo json_encode(['error' => 'Invalid signature']);
    exit;
}

$orderId = $webhookData['order_id'];
$status = $webhookData['payment_status'];

switch ($status) {
    case 'paid':
    case 'overpaid':
        break;
    case 'underpaid_check':
        break;
    case 'underpaid':
        break;
    case 'cancel':
        break;
}

http_response_code(200);
echo json_encode(['success' => true]);
?>

Important

Best Practices:

- Idempotency: Always check if the payment has already been processed (by order_id).

- Async: Webhooks may arrive multiple times. Handle them idempotently.

- Fast response: Return HTTP 200 as quickly as possible.

- Retries: If we don't receive HTTP 200, the webhook is resent after 2 minutes. Maximum 5 retry attempts.

- Security: ALWAYS verify the sign signature before processing.

?>

Static Wallet Webhook Fields
Field	Description
uuid	Transaction (invoice) UUID
order_id	Your static wallet order_id
amount	Amount in USD
currency	Wallet currency (USDT, BTC, etc.)
merchant_amount	Amount after fee deduction
payment_status	Payment status (always 'paid' for statics)
txid	Blockchain transaction hash
sign	HMAC-SHA256 signature for verification
Important

Recommendations:

- Unique order_id: Use a unique order_id for each user/order.

- Idempotency: Check txid before processing to avoid duplicates.

- Security: ALWAYS verify the sign signature before crediting funds.

- Fees: Use merchant_amount for crediting, not payment_amount.
